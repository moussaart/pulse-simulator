AI Training Environment

Integrate Reinforcement Learning agents and collect bulk simulation data.

Architecture Overview

The PULSE simulator provides a dedicated AI training environment with a clear separation between the main simulation and AI training logic. The architecture uses a TCP/IP socket for communication between the simulator and external RL agents, plus a programmatic API for configuration and data retrieval.

Online AI Training Client

The external_ai_client project folder contains the files needed to train a Reinforcement Learning agent (using OpenAI Gym / Gymnasium and PyTorch) that interacts with the PULSE Simulator.

1. The Server (Inside PULSE Simulator)

We added a custom algorithm called AI Gym Server (Port 5555) directly into the simulator's code. When you select this algorithm in the GUI, it starts listening for an AI Agent to connect.

2. The Client (external_ai_client folder)

The pulse_rl_env.py creates a standard gymnasium.Env and connects to the GUI via a TCP socket. It receives the current measurements and true positions (for rewards), sends back the chosen anchors (action), and lets the GUI do the heavy mathematically filtering (EKF) using only the selected anchors.

Setup & Training

  1. Install the basics for your AI environment: 
    pip install gymnasium numpy
  2. Open the PULSE Simulator GUI.
  3. Go to the Algorithms selection panel and choose AI Gym Server (Port 5555).
  4. Click Start to run the simulation scenario. It will wait for the AI to connect.
  5. Open a terminal in the external_ai_client folder and run: 
    python train_agent.py

As the AI Agent plays, the GUI updates dynamically. The AI is rewarded for selecting Line-of-Sight (LOS) anchors and penalized for using Non-Line-of-Sight (NLOS) anchors.

API Reference

AITrainingAPI — Configuration & Data Access

The primary facade for AI training. Imported from src.api:

from src.api import AITrainingAPI

api = AITrainingAPI(sim_context=app)

Scope Isolation

All SET operations modify a training-local configuration copy. The main simulation state is never affected.

AIGymServer — RL Communication

TCP server for OpenAI Gym-style interaction with external agents:

from src.api.ai_gym_server import AIGymServer

server = AIGymServer(host="localhost", port=5555)
server.start()

TrainingDataAPI — Offline Data Collection

For batch data collection and export (dataset creation):

from src.api import TrainingDataAPI

api = TrainingDataAPI(buffer_size=10000)
api.select_data(channel=True, filter_outputs=True, ground_truth=True)
api.enable_collection()
# ... run simulation ...
api.export_to_file("training_data.npz")
